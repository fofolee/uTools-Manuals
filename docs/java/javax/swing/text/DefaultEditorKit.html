    public class DefaultEditorKit
extends EditorKit
    这是文本组件所需的一组事物，作为某种类型的文本文档的合理功能的编辑器。 此实现提供了将文本视为纯文本并为简单编辑器提供最小的操作集的默认实现。
    新线
    有两个处理换行符的属性。 系统属性line.separator被定义为与平台相关的“\ n”，“\ r”或“\ r \ n”。 还有一个在DefaultEditorKit中定义的DefaultEditorKit ，称为EndOfLineStringProperty ，它是在文档加载时自动定义的，这是首次出现任何换行符。 加载文档时， EndOfLineStringProperty被正确设置，当文档被写回时，使用EndOfLineStringProperty 。 但是当文档在内存中时，“\ n”字符用于定义换行符，无论文档在磁盘上如何定义换行符。 因此，为了搜索目的，应始终使用“\ n”。 创建新文档时，尚未定义EndOfLineStringProperty ，则在写入文档时将使用System属性。

    请注意， EndOfLineStringProperty是使用get/putProperty方法设置在Document上。 子类可能会覆盖此行为。

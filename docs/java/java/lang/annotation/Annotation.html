    public interface Annotation
    所有注释类型扩展的公共接口。 注意，手动扩展这个接口不限定注释类型。 还要注意，此接口本身并不定义注释类型。 有关注释类型的更多信息，请参见The Java™ Language Specification的第9.6 节 。 AnnotatedElement接口讨论了将注释类型从不可重复性转变为可重复时的兼容性问题。

    1.5

    • 方法详细信息

      • equals

        boolean equals(Object obj)
        如果指定的对象表示在逻辑上等同于该注释的注释，则返回true。 换句话说，如果指定的对象是与该实例相同的注释类型的实例，则返回true，所有这些实例的所有成员都等于此注释的相应成员，定义如下：
        • 如果x == y ，其值为x和y的两个对应的原始类型的成员被认为是相等的，除非它们的类型是float或double 。
        • 两个对应float个成员，其值是x和y都认为，如果Float.valueOf(x).equals(Float.valueOf(y))相等。 （与==运算符不同，NaN被认为等于自己， 0.0f不等于-0.0f ））
        • 两个对应double个成员，其值是x和y都认为，如果Double.valueOf(x).equals(Double.valueOf(y))相等。 （与==运算符不同，NaN被认为等于自己， 0.0不等于-0.0 ）
        • 两个相应的String，Class，enum或annotation类型的成员的值是x和y被认为是相等如果x.equals(y)。 （请注意，此定义对于注释类型的成员是递归的。）
        • 两个相应的数组类型的成员和x y被视为相等如果Arrays.equals(x, y)中，出于适当重载Arrays.equals(long[], long[]) 。
        重写：
        equals在类别 Object
        参数
        obj - 与之比较的参考对象。
        结果
        如果指定的对象表示逻辑上等同于该注释的注释，则为true，否则为false
        另请参见：
        Object.hashCode() ， HashMap
